Passwords


What is a password? And why does it matter?

A password is a (usually) secret combination of characters (letters, numbers, r@nd0m symb01s) that is used to authenticate a user's identity, and therefore provide controlled access to services. Passwords ensure that only authorized individuals can gain access to protected information.

So, if a password falls into the hands of the wrong person, they can falsely authenticate themselves as the intended user, gain access to confidential information, steal someone's identity, all the money to their name, and so much more...

You may be wondering; how do I keep my password secure? Here's some guidelines from the National Institute of Standards and Technology (NIST):

  1. Avoid adding complexity using l33t speak, or adding numbers to some 'base' word
    • This barely makes it harder for a hacker to get to you through a Brute-Force Attack
    • It's harder to memorize for the user (you)
  2. Use at least 8 characters, but the longer the password, the better it is.
    • If the number of possible characters is 94 (26 upper-case + 26 lower-case + 10 digits + 32 symbols) then 948 different passwords are possible (for an 8-character password!). The password has 52-bit entropy.
      • Bits of Entropy: the strength of the password is defined by the number of attempts required to successfuly guess it.
        If a password has n-bit entropy then it would take 2n tries to crack it!

    The password "SaltedHashbrowns1" is 17 characters long. It uses upper-case, lower-case and numerical digits. The character set is (26 upper-case + 26 lower-case + 10 digits) = 62 characters. The number of guesses to crack the password will be 6217, or 2101.2 with 101.2 bits of entropy

    Hint: the entropy is calculated by \(e = L(log(C)/log(2))\). L is the password length, C is the size of character set

  3. Avoid picking a word. It's better to mix characters randomly. Use different characters: upper-case, lower-case, numbers, symbols, Unicode...

    4. The passwords "SaltedHashbrowns1" and "A8eF0nd#j%lX*v$T" are both 17 characters long, and have the same 101.2 bit-entropy. But, the latter would be guessed much later in an attack.

  4. Don't use any passwords that are commonly known (maybe you'll find your password here ), as well as context-specific words (related to the you or the service, such as your username, the name of the service, etc). It's better to just not use words!
  5. If asked to use a hint or secret question, use gibberish! Your mother's maiden name is not a secret, neither is your dog's first name...
  6. Use a Password Manager, and Multi-Factor Authentication (MFA)
    • Password Managers are a secure platform for master-storage of your passwords, controlled by a master password
    • MFA increases security as it involves using more than one authentication mechanism to verify a user's identity. This can be done by the service asking for a PIN, or facial recognition along with your password to ensure it is really you entering the account!

You may be wondering, how can someone even steal my password? How would they know my first pet's name? Well here's how!


Common Password Attacks

  1. Brute Force Attack: the hacker runs a script which attempts every possible password, one-by-one
  2. Keylogging: the hacker secretly implements a keylogger on your device, which is a tool that records your typing activity
  3. Dictionary Attack: generating a list of common passwords and trying different combinations on them. For example, trying Winter2020, then Winter2021 and so on...
  4. Phishing Attack: this involves a hacker psychologically manipulating someone to disclose their username and password, by pretending to be a trusted entity. It is typically done through scam emails that take you fake websites. Be careful!

Hashes


Let's get to the core of Salted Hashbrowns. We have to first start with the Hashes

Passwords cannot be stored in readable text (plaintext) , they must be stored in gibberish (ciphertext). Otherwise, anyone can easily steal your password file on your computer and read all of your passwords! The process of converting readable text into gibberish is called encryption, and decryption is the reverse of encryption (converting gibberish to readable text)

Encryption: "hello" becomes "82efn"

Decryption: "82efn" returns to "hello"


A way to encrypt passwords is called hashing. But hashing only encrypts the data, which cannot then be decrypted. Instead of storing the password, we store the hash, which comes from a hash function. So we are storing f(x) = H(password) or the output from putting your password into the hash function.

But what is a hash function?

A function where you enter plaintext, it stirs it up, and you get gibberish. It turns the message into a short, unique, readable string. There are many hash functions; SHA-256 (Secure Hash Algorithm 256-bit, which outputs a 256-bit hash) is very commonly used. Another popular one is MD5 (Message Digest 5, with a 128-bit hash output)

Salts


Onto the other half, we'll now discuss the Salt!

Since many people like to choose bland passwords, (like your own name, your pet's name, and so on) hashing a password isn't enough.
Therefore, when a password hash is stored (remember, NOT the password, but the hash), a salt is added for some extra security. So our typical process becomes:
H(password) → H(password + salt)
The salt is a large, randomly generated string which is different for each user. So even if two people have the same password, which creates the same hash, the salt will be different and so will the overall encryption.

Jack's password is BurntHashbrown

His salt is a29enth3s9kr14n

So the salted hash of his password is H("BurntHashbrown a29enth3s9kr14n") = "d9gne9fne8d2h4k"

Now Jill could have the same password, BurntHashbrown , but her salt would be completely different, resulting in a totally different encryption.


Isn't that so smart?! One would think... but let's not jump to conclusions just yet. Salting passwords can slow some approaches to hacking, but not others. We can slow the hackers down by iterating hash functions:
H(H(H(H(H(H...H(password)))))) This increases security and can slow hackers by magnitudes!
